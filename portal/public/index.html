Deploy an environment

    ✅ Environment is live:
    Deploy ≈ 45 min (AEM boot dominates). Stage/prod pause for reviewer approval in GitHub. Destroy ≈ 10 min.

    1 · Credentials

    Held in memory only — re-enter after a portal restart. Delete the AWS key in IAM once setup is done.

    2 · AWS account plumbing (once per account)

    Creates the OIDC provider, deploy roles, state bucket, seed bucket and budget alarm by running the bootstrap-aws workflow. The AWS key is placed in repo secrets only for the duration of the run, then removed.

    3 · Licensed AEM binaries → your private bucket

    You must own an AEM 6.5 license. Files go to your private S3 seed bucket — never into git. Upload files or paste direct-download URLs.

    4 · Wire the pipeline variables

    Sets BINARIES_SEED_BUCKET and the per-environment AWS_ROLE_ARN automatically. Optionally set the admin password used when deploying with hardening.
